Appendix B — Data Pipeline & Join Logic

Geographic Join Strategy

Food Businesses (8,588)
  ├── lat/lon → zip code → Census income/poverty
  ├── lat/lon → tract FIPS → CDC health outcomes
  ├── address → street norm → Crime incidents
  └── business_id → inspections → violations

Crime Reports (259,660)
  ├── address → "NNN Block STREET" → street norm
  └── street norm → matched to food biz streets

Result: unified risk profile per zip code
  ├── food safety: violations, Red count, critical count
  ├── crime: incidents on food streets
  ├── economic: poverty rate, median income
  └── health: diabetes, obesity, food insecurity

Appendix C — Agent Specifications

SENTINEL — Fast Triage

Model: Nemotron Nano (3B active, MoE)
Input: All datasets, pre-computed risk scores
Output: Flagged patterns with preliminary severity (1-5)
Decision: "Is this worth investigating?" — filters noise from signal
Latency target: <5s per scan
Trigger: Nightly batch + real-time on new inspection results

ANALYST — Deep Reasoning

Model: Nemotron 70B (full, 128K context)
Input: Sentinel flags + full cross-silo data for flagged area
Output: Severity assessment, root cause hypothesis, affected population, recommended intervention
Decision: "How bad is this? Why? Who's affected?"
Latency target: 30-60s per analysis
Context: Fed zip-level stats, top violators, health outcomes, crime patterns, census demographics

ADVISOR — Action Generator

Model: Nemotron 70B
Input: Analyst findings + recipient profile
Output: Tailored action plan per recipient type
Decision: "What should this specific person DO?"
Output types:
  • Health Officer: strategic brief with intervention plan
  • Supervisor: prioritized queue with pre-visit briefs
  • Owner: plain-language checklist with specific fixes
  • Resident: conversational safety guidance
  • Elected official: scorecard with talking points

MESSENGER — Routing & Delivery

Model: Nemotron Nano
Input: Advisor outputs + urgency level + routing rules
Output: Formatted, delivered messages
Decision: "Who gets this, when, and how?"
Channels: Dashboard, email digest, SMS alert, PDF report
Tracking: Delivery confirmation, open/read status, follow-up scheduling

Appendix G — Compounding Risk Score Methodology

Composite Risk Score

Each zip code is ranked on 4 dimensions. The composite score is the weighted sum of percentile ranks:

risk_score =
  avg_critical_violations.rank(pct) × 25
+ poverty_rate.rank(pct) × 25
+ crime_per_business.rank(pct) × 25
+ total_red_inspections.rank(pct) × 25

Range: 0–100. Threshold for alert: 70+

Equal weighting ensures no single dimension dominates. The power is in the combination — a zip must be failing across multiple dimensions to trigger an alert.

Why OpenClaw, not sequential API calls
Structured context accumulation
Each agent's output is typed and injected into the next agent's prompt as structured data — not raw text. The Analyst knows exactly what the Sentinel decided and why.
Decision-driven routing, not hard-coded sequence
Severity < 3 → Analyst skipped. Severity = 5 → Messenger routes immediately, not on schedule. OpenClaw reads the payload and adjusts the pipeline at runtime.
128K context window = full cross-silo profile in one pass
Nemotron 70B on the GB10's 128GB unified memory holds the complete ZIP profile — all 12 metrics, benchmarks, top violators — without truncation. The Analyst reasons over the whole picture.

Appendix I — Why Agents, Not Scripts

A script can join datasets. Only agents can reason, decide, and act.

Without agents (just a script)

Static thresholds

Hard-coded rules: "if violations > 10, flag." No reasoning about WHY or whether it matters in context.

One-size-fits-all output

Same report for everyone. A health officer and a restaurant owner see the same data dump.

No adaptation

Can't explain its reasoning. Can't adjust when a pattern doesn't fit the rules. Breaks on edge cases.

With OpenClaw agents

Contextual reasoning

Analyst UNDERSTANDS why 95122 is different: "High poverty + food insecurity means these failures disproportionately harm vulnerable residents."

Tailored communication

Advisor generates different outputs for each recipient: strategy for officials, checklists for owners, plain language for residents.

Autonomous decision-making

Sentinel decides what's worth flagging. Analyst decides severity. Advisor decides what to recommend. Messenger decides urgency.
